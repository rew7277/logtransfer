🔴 CRITICAL BUG FIXES
═══════════════════════════════════════

1. FIXED: 500 Error on POST /login
   File: app.py  (login route ~line 1978)
   Root cause: The SQL query for login was:
     SELECT u.id, u.name, u.email, u.role, u.org_id, u.password_hash,
            u.email_verified, o.admin_only
     FROM users u JOIN organizations o ON o.id = u.org_id ...
   It did NOT include o.slug or o.public_id, but the login response
   tries to access user["slug"] → IndexError: No item with that key → HTTP 500.
   Fix: Added o.slug, o.public_id to the login SELECT.

2. FIXED: withEnv() function was called but never defined
   File: templates/index.html
   withEnv() is called in 4 places (bootstrap, sources, errors, uploads)
   but the function body was missing entirely → ReferenceError in browser.
   Fix: Added full withEnv() implementation.

3. FIXED: setEnvironment() function was called from the env select
   but never defined — switching environments silently did nothing.
   Fix: Full setEnvironment() implementation that refreshes all data panels.

4. FIXED: togglePw() in login.html referenced getElementById('eyeIcon')
   which does not exist → JS error on password show/hide toggle.
   Fix: Rewrote togglePw() to use querySelector('.pw-toggle').

5. FIXED: Overview h1 had broken HTML — closed with </div> not </h1>
   causing malformed DOM and visual layout corruption.
   Fix: Replaced with new overview-hero component.
