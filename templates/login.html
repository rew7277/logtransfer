Platform features

Everything your website should explain before the user clicks Create Account.

Each organization can sign in, manage access, connect data sources, and monitor logs from a clean enterprise workspace.

📂

Upload any log files

Users can upload application logs, plain text logs, multiline stack traces, and structured files without breaking event grouping.

☁️

AWS S3 integration

Test buckets, save credentials, attach prefixes, and use them as a scheduled ingestion source for archived or live operational logs.

🔌

Remote API ingestion

Connect any log-producing API endpoint with optional bearer tokens and JSON headers, then validate it before saving.

⏱️

Scheduled ingestion jobs

Create cron-based jobs for S3 and API polling, keep them paused or scheduled, and trigger run-now actions from the UI.

🚨

Alert rules

Define error spikes, SLA breach rules, stalled-ingestion detection, and channel-based escalation using Slack, email, or PagerDuty.

👥

Organization users

Admins can create users inside the same organization with roles like admin, manager, developer, and tester.

🛡️

Admin-only mode

If you want everything controlled only by admins, switch the workspace to admin-only and block non-admin access.

🎨

Theme & branding

Choose black, white, or green themes, set organization identity, and prepare for org subdomains like company.vewit.com.

📜

Audit trail

Track logins, user creation, integration changes, job updates, and other admin actions for better accountability.
